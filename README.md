## вңҰ What Is This?

A **web application** that generates **antique rug knot-grid patterns** вҖ” the same kind of chart a real rug weaver uses to hand-knot a rug. Every dot in the output grid represents one physical knot.

Designs are **procedurally generated** using authentic antique rug structural grammars вҖ” field layouts, vine scrolls, floral rosettes, palmettes, central medallions, and tiered borders вҖ” all placed with weaving-aware constraints.

## вҡҷпёҸ How It Works

### 1. Вј Generation + Mirror Symmetry

Most antique rugs are **bi-axially symmetric** вҖ” left/right and top/bottom. The engine:

1. Generates **one quadrant** (top-left Вј of the rug)
2. Reflects it to fill all 4 quadrants
3. Adds **border + medallion** on the full grid (centered)
4. Applies optional **variation** (subtle abrash-like imperfection)

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ  Q  в”Ӯ вҶ’Q  в”Ӯ   Q = generated quadrant
в”ңв”Җв”Җв”Җв”Җв”Җв”јв”Җв”Җв”Җв”Җв”Җв”Ө   вҶ“Q = vertical mirror
в”Ӯ вҶ“Q  в”ӮвҶ“вҶ’Q  в”Ӯ   вҶ’Q = horizontal mirror
в””в”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”ҳ   вҶ“вҶ’Q = both
```

### 2. Structural Spec (`.rug.json`)

Every design is stored as a **structured project file**, not just pixels:

```json
{
  "id": "uuid",
  "name": "My Sarouk",
  "style": "sarouk",
  "seed": "abc123",
  "symmetry": "mirror-both",
  "variation": 0.08,
  "palette": { "colors": [...] },
  "layout": {
    "border": { "motif": "vine-scroll", "tileRepeatX": 8, ... },
    "medallion": { "sizeX": 0.38, "innerRings": 2, ... },
    "field": { "vineDensity": 0.65, "leafRoundness": 0.7, ... }
  },
  "locks": [],
  "history": [...]
}
```

This means **every iteration is a diff** вҖ” not a new image from scratch.

### 3. Deterministic Seeds + Scoped Seeds

- **Same seed + same params = identical output** every time
- Change one parameter вҶ’ only that part changes
- `scopedRNG(seed, "field")` keeps field consistent when you change only the border seed

### 4. AI Mode (OpenRouter)

The LLM never outputs pixels. It outputs **patch operations** on the spec:

```
You: "make the leaves rounder and less dense"

AI вҶ’ ops:
  { op: "set", path: "layout.field.leafRoundness", value: 0.9 }
  { op: "set", path: "layout.field.vineDensity",   value: 0.4 }

Renderer: spec + ops вҶ’ new knot grid
```

## рҹӨ– AI Integration

The app connects to [OpenRouter](https://openrouter.ai) вҖ” a unified API for many LLMs including Claude, GPT-4, Gemini, and open-source models.

**How to use:**
1. Get a key at [openrouter.ai/keys](https://openrouter.ai/keys)
2. Enter it in the **AI tab** inside the app
3. Type a change request вҖ” the AI patches only what you describe

**The AI understands things like:**
- "make the border pattern more geometric"
- "reduce leaf density, keep the medallion"
- "make it look more worn / more variation"
- "increase flower complexity, bigger rosettes"
- "remove the vines, keep everything else"
- "make the border 20% thinner"

The key is **never stored** вҖ” it's sent per-request to OpenRouter.
